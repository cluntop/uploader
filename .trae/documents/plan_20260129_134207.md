# 为项目添加 Docker 支持

## 1. 创建 Dockerfile 文件

使用多阶段构建策略：

- **第一阶段**：使用 Node.js 镜像安装依赖并构建项目
- **第二阶段**：使用轻量的 Node.js Alpine 镜像，安装 `serve` 包并将构建产物复制到容器中，使用 `serve` 命令提供静态文件服务

## 2. 创建 docker-compose.yml 文件

配置以下内容：

- 服务名称、镜像名称、容器名称
- 端口映射（将容器的 3000 端口映射到主机的 8000 端口）
- 重启策略
- 环境变量配置（如果需要）

## 3. 更新 README.md 文件

添加 Docker 使用说明，包括：

- 构建 Docker 镜像的命令
- 运行 Docker 容器的命令
- 使用 docker-compose 运行的命令
- 环境变量配置说明

## 4. 构建和测试

- 构建 Docker 镜像
- 运行 Docker 容器
- 测试服务是否正常访问

## 5. 优化

- 使用官方的 Node.js 镜像
- 最小化 Docker 镜像大小
- 确保容器安全运行
- 提供合理的默认配置
