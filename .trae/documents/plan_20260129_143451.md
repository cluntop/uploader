# 修改 docker-compose.yml 文件

## 修改内容

1. **构建配置**：
   - 使用简化的 `build: .` 配置

2. **镜像配置**：
   - 使用 `node:18-alpine` 作为基础镜像

3. **命令配置**：
   - 使用 `/bin/sh -c "npm run build && npm install -g serve"` 作为命令
   - 注意：这个命令需要在容器启动时执行完整的构建和安装过程

4. **工作目录**：
   - 设置工作目录为 `/app`

5. **容器名称**：
   - 将容器名称改为 `emos`

6. **重启策略**：
   - 设置重启策略为 `unless-stopped`

7. **端口映射**：
   - 保留 `8000:3000` 的端口映射

8. **环境变量**：
   - 保留现有的环境变量配置

## 注意事项

- 使用 `node:18-alpine` 作为基础镜像，容器会在启动时执行完整的构建过程
- 这可能会导致容器首次启动时间较长
- 需要确保项目的依赖管理正确，以便在容器中成功构建

## 预期效果

修改后的 docker-compose.yml 文件将使用与 Dockerfile 类似的构建过程，但通过 docker-compose 的配置来实现，使配置更加简化和直接
