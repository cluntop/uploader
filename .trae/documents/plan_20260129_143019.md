# 修改 docker-compose.yml 文件

## 修改内容

1. **构建配置**：
   - 将详细的构建配置改为简化的 `build: .`

2. **镜像配置**：
   - 将镜像名称改为 `node:18-alpine`

3. **命令配置**：
   - 添加命令 `/bin/sh -c "npm run build && npm install -g serve"`

4. **工作目录**：
   - 添加工作目录配置 `working_dir: /app`

5. **容器名称**：
   - 将容器名称改为 `emos`

6. **保留配置**：
   - 保留端口映射 `8000:3000`
   - 保留重启策略 `unless-stopped`
   - 保留环境变量配置

## 注意事项

- 使用 `node:18-alpine` 镜像意味着容器会在启动时执行构建和安装 serve
- 这种方式与当前的多阶段构建 Dockerfile 不同，可能会导致容器启动时间较长
- 需要确保项目的依赖管理正确，以便在容器中成功构建

## 预期效果

修改后的 docker-compose.yml 文件将使用 node:18-alpine 镜像，在容器启动时执行构建和安装 serve，然后运行服务。
